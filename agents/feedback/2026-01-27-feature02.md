# 2026-01-27 — Feature 02 (Stripe Billing) Decisions

## Décisions techniques
- Plans seedés en DB (FREE/PREMIUM/VIP) avec `stripe_price_id` placeholders (`price_premium_eur`, `price_vip_eur`) à remplacer en prod.
- Webhook Stripe idempotent via `webhook_events` (event_id unique) + raw body obligatoire.
- Attribution rôles premium/vip via webhook uniquement (role_grants + update `users.roles`).
- Quotas mensuels alignés sur les périodes Stripe (period_start/end).
- `/billing/checkout-session` protégé par rate‑limit user+IP + captcha adaptatif (sur échecs).
